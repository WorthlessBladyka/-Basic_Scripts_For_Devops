## Flags
### --cpuset-cpus
```
$ docker run -d --cpuset-cpus 0 --cpu-shares 768 benhall/stress
			# контейнер работает только с конкретным ядром
			# не может использовать другие ядра процессора
```
### --cpu-shares
```
$ docker run --cpuset-cpus 0 --cpu-shares 256 benhall/stress
			#устанавливает относительный вес среди других запущенных контейнеров
```
```
# Контейнер A: --cpu-shares 1024
# Контейнер B: --cpu-shares 512
# Контейнер C: --cpu-shares 512

Общая сумма shares = 1024 + 512 + 512 = 2048

Доли CPU:
A: 1024 / 2048 = 50%  (1024 ÷ 2048 = 0.5)
B: 512 / 2048  = 25%   (512 ÷ 2048 = 0.25)  
C: 512 / 2048  = 25%   (512 ÷ 2048 = 0.25)
```
### benhall/stress
```
# Специализированный образ для тестирования нагрузки
# Создает искусственную нагрузку на CPU/RAM
```
### --net
```
--net bridge      # По умолчанию (мостовая сеть)
				  # Создает изолированную сеть через Docker bridge с NAT
--net host        # Сетевой namespace хостa
				  # Контейнер использует сетевой stack хоста напрямую, без изоляции
--net none        # Без сети (полная изоляция)
--net container   # Использовать сеть другого контейнера
--net custom-net  # Пользовательская Docker сеть
```
### --pid 
```
--pid container:<name|id>  # Разделять PID namespace с другим контейнером
--pid host                 # Использовать PID namespace хоста
# По умолчанию: изолированный PID namespace
```
!Контейнер имеет собственную нумерацию процессов, изолированную от хоста.
