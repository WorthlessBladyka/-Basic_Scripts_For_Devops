## Практические сценарии

### Когда использовать вертикальное масштабирование:
```
- Транзакционные БД (MySQL, PostgreSQL)
- Приложения с сложными вычислениями
- Начальные этапы проекта
- Когда простота важнее масштабируемости

 Пример: Финансовая система банка
- Сложные транзакции
- Требования ACID
- Проще управлять одним сервером
```
### Когда использовать горизонтальное масштабирование:
```
1. Веб-приложения с высокой нагрузкой
2. Системы с пиковыми нагрузками
3. Big Data и аналитика
4. Когда нужна высокая доступность

Пример: Социальная сеть
- Миллионы пользователей
- Периоды высокой нагрузки
- Требуется 99.9% uptime
```
## Реляционные БД (SQL)

- **Структура:** Таблицы с строго определенными столбцами
- **Схема:** Жесткая, требует предварительного определения
- **Целостность данных:** ACID свойства
- **Примеры:** MySQL, PostgreSQL, MariaDB, Oracle

## Нереляционные БД (NoSQL)

- **Структура:** Различные модели (документы, ключ-значение, графы, колоночные)
- **Схема:** Гибкая, может изменяться динамически
- **Масштабируемость:** Горизонтальное масштабирование
- **Примеры:** MongoDB, Redis, Cassandra, Neo4j

## ACID

==ACID== — это набор свойств, которые гарантируют надежность транзакций в базах данных.
- Atomicity — Атомарность
- Consistency — Согласованность
- Isolation — Изолированность
- Durability — Долговечность
### Atomicity
"Все или ничего" — транзакция выполняется полностью либо не выполняется вообще.
### Consistency
"Данные всегда в валидном состоянии" — транзакция переводит базу из одного согласованного состояния в другое.
- Все ограничения (constraints) соблюдаются
- Данные всегда соответствуют бизнес-правилам
### Isolation
"Транзакции не мешают друг другу" — параллельные транзакции выполняются так, как будто они выполняются последовательно.
### Durability
"Раз записал — навсегда" — после подтверждения транзакции изменения сохраняются даже при сбое системы.

### **✅ Области применения:**
- **Банковские системы** (переводы, платежи)
- **Финансовые операции** (биржевые торги)
- **Системы бронирования** (авиабилеты, отели)
- **Инвентаризация** (учет товаров)
- **Медицинские системы** (записи пациентов)
### **❌ Когда можно ослабить (BASE):**
- **Социальные сети** (лайки, комментарии)
- **Кэширование**
- **Аналитика** (данные могут быть немного устаревшими)
- **Системы сбора метрик**
