## pwd 

pwd

Вывести полный путь до текущей рабочей директории, в которой находится пользователь.

Команду pwd часто используют в bash скриптах для присвоения переменной пути до рабочего каталога.

Пример использования команды pwd в скриптах для присвоения переменной пути до текущего каталога:

```
DIR=`pwd`
или
DIR=$(pwd)
```

## ls

$ ls опции /путь/к/folder

Посмотреть, что в папке и посмотреть расширение файлов

`~`  — home folder;

`.` — указывает на директорию, в которой вы находитесь в данный момент;

`..` — обозначает родительскую директорию.

- **-a - отображать все файлы, включая скрытые, это те, перед именем которых стоит точка;**

- **-A** - не отображать ссылку на текущую папку и корневую папку . и ..;
- **--author** - выводить создателя файла в режиме подробного списка;
- **-b** - выводить Escape последовательности вместо непечатаемых символов;
- **--block-size** - выводить размер каталога или файла в определенной единице измерения, например, мегабайтах, гигабайтах или килобайтах;
- **-B** - не выводить резервные копии, их имена начинаются с ~;
- **-c** - сортировать файлы по времени модификации или создания, сначала будут выведены новые файлы;
- **-C** - выводить колонками;
- **--color** - включить цветной режим вывода, автоматически активирована во многих дистрибутивах;
- **-d** - выводить только директории, без их содержимого, полезно при рекурсивном выводе;
- **-D** - использовать режим вывода, совместимый с Emacs;
- **-f** - не сортировать;
- **-F** - показывать тип объекта, к каждому объекту будет добавлен один из специализированных символов */=>@|;
- **--full-time** - показывать подробную информацию, плюс вся информация о времени в формате ISO;
- **-g** - показывать подробную информацию, но кроме владельца файла;
- **--group-directories-first** - сначала отображать директории, а уже потом файлы;
- **-G** - не выводить имена групп;

- **-h - выводить размеры папок в удобном для чтения формате;**

- **-H** - открывать символические ссылки при рекурсивном использовании;
- **--hide** - не отображать файлы, которые начинаются с указанного символа;
- **-i** - отображать номер индекса inode, в которой хранится этот файл;

- **-l - выводить подробный список, в котором будет отображаться владелец, группа, дата создания, размер и другие параметры;**

- **-L** - для символических ссылок отображать информацию о файле, на который они ссылаются;
- **-m** - разделять элементы списка запятой;
- **-n** - выводить UID и GID вместо имени и группы пользователя;
- **-N** - выводить имена как есть, не обрабатывать контролирующие последовательности;
- **-Q** - брать имена папок и файлов в кавычки;
- **-r** - обратный порядок сортировки;
- **-R** - рекурсивно отображать содержимое поддиректорий;
- **-s** - выводить размер файла в блоках;

- **-S - сортировать по размеру, сначала большие;**
- **-t - сортировать по времени последней модификации;**

- **-u** - сортировать по времени последнего доступа;
- **-U** - не сортировать;

- **-X - сортировать по алфавиту;**

- **-Z** - отображать информацию о расширениях SELinux;
- **-1** - отображать один файл на одну строку.

## cd

## clear and reset

$ clear

Команда "clear" - удаляют весь видимый текст с экрана терминала.
Альтернатива CTRL + L.

$ reset

Команда «reset» в Linux используется для повторной инициализации текущего терминала до известной конфигурации по умолчанию. Это особенно полезно, когда сеанс терминала искажается, перестаёт отвечать или демонстрирует странное поведение из-за неправильно настроенных программ.

## cat - объединение файлов

```
$ cat опции файл1 файл2 …
```

1. -b – включить нумерацию непустых строк (заполненных данными).
2. -E – в конце каждой строки будет отображаться символ $.
3. -n – пронумеровывать все строки, даже пустые (без данных).
4. -s – автоматически удалять пустые повторяющиеся строки.
5. -T – помечать табуляции комбинацией символов ^I.
6. -h – вывести на экран монитора справочную информацию.
7. -v – отобразить текущую версию утилиты.

Отображает содержимое указанного файла:

```
$ cat file
```

Если нужно передать системе несколько файлов, достаточно перечислить их в параметрах:

```
$ cat file1 file2 file3
```

Если требуется удалить пустые строки:

```
$ cat -n -b file1
```

Чтобы завершить запись, нужно нажать <Ctrl+D>. Можно направить данные в указанный файл на диске:

```
$ cat > file1

$ cat file1
```
Запись данных из нескольких файлов в один:

```
$ cat file1 file2 > file3

$ cat file3
```

## find - найти файл
```
$ find directory-to-search criteria action
```
- directory-to-search (каталог поиска)
- criteria (критерий) – критерий, по которым нужно искать файлы;
- action (действие) – что делать с каждым найденным файлом;

**Пример:**
	$ find . -name "*.txt"

**Критерий:**
- -name "file*.txt" - искать по имени
- -type d - искать по типу
	- f – простые файлы;
	- d – каталоги;
	- l – символические ссылки;
	- b – блочные устройства (dev);
	- c – символьные устройства (dev);
	- p – именованные каналы;
	- s – сокеты;
- -size +1G - искать по размеру(искать файлы с размером более 1гб)
	- "+" — Поиск файлов больше заданного размера
	- "-" — Меньше заданного размера
	- Отсутствие знака, размер файлов в поиске должен совпадать.
		- c — Байт
		- k — Кбайт
		- M — Мбайт
		- G — Гбайт
и др

**Действия:**
- delete — Удаляет соответствующие результатам поиска файлы
- -ls — Вывод более подробных результатов поиска с:  
    - Размерами файлов.
    - Количеством inode.
- -print Стоит по умолчанию, если не указать другое действие. Показывает полный путь к найденным файлам.
- -exec Выполняет указанную команду в каждой строке результатов поиска.
## locate - искать файл по имени
**locate**— искать файл по имени или фрагменту имени. 

```
locate <имя или часть имени>
```
### Краткая суть

- **`locate`** — **Поиск по базе данных**. Быстрый, но ищет только по имени и использует заранее собранную базу данных, которая может быть неактуальной.
- **`find`** — **Поиск в реальном времени**. Медленнее, но мощнее. Ищет в файловой системе прямо сейчас, используя множество критериев (имя, размер, время, права и т.д.).

**Совет:** Если вы только что создали файл и `locate` его не находит, обновите базу данных вручную: `sudo updatedb`.

## touch - (create new file and обновление времени файла)

### Создание пустых файлов

```
    touch newfile.txt
```

Можно создать несколько файлов одновременно:
```
    touch file1.txt file2.txt file3.log
```

### Обновление временных меток файлов

Eсли файл уже существует, `touch`обновляет его временные метки доступа и изменения до текущего системного времени по умолчанию. Это может быть важно для таких задач, как запуск резервного копирования, зависящего от времени изменения файла, или просто для обновления информации о последнем изменении файла.

### Ключевые параметры:

- `-a`: Изменяет только время доступа.
- `-m`: Изменяет только время модификации.
- `-c`: предотвращает создание нового файла, если он еще не существует (выдает сообщение об ошибке, если файл не найден).
- `-t [[CC]YY]MMDDhhmm[.ss]`: Устанавливает время доступа и изменения на определенную дату и время вместо текущего времени.
    - `CC`: Век (первые две цифры года).
    - `YY`: Год (последние две цифры года).
    - `MM`: Месяц (01-12).
    - `DD`: День (01-31).
    - `hh`: Час (00-23).
    - `mm`: Минута (00-59).
    - `ss`: Второй (00-59).
- `-d STRING`: устанавливает время доступа и изменения, используя понятную человеку строку даты (например, «вчера», «следующий понедельник», «06.10.2025 10:30»).

Пример установки конкретной временной метки:
```
touch -t 202412251030.00 christmas_plan.txt
```
return: эта команда устанавливает временную метку `christmas_plan.txt`на 25 декабря 2024 года, 10:30:00.
## sort - сортирует

```
$ sort file.txt

$ sort параметр file2.txt
```
**Параметры:**
- **-b** - не учитывать пробелы
- **-d** - использовать для сортировки только буквы и цифры
- **-i** - сортировать только по ASCII символах
- **-n** - сортировка строк linux по числовому значению
- **-r** - сортировать в обратном порядке
- **-с** - проверить был ли отсортирован файл
- **-o** - вывести результат в файл
- **-u** - игнорировать повторяющиеся строки
- **-m** - объединение ранее отсортированных файлов
- **-k** - указать поле по которому нужно сортировать строки, если не задано, сортировка выполняется по всей строке.
- **-f** - использовать в качестве разделителя полей ваш символ вместо пробела.

## cut - режет файлы

```
$ cut -d ">" -f 3 file.txt
```
! Используется для извлечения определенных разделов из строк файлов, которые могут быть разделены по байтам, символам или полям.
! В примере, показано, что у нас указан делиметр ">"  с позицией 3. То есть, начиная с 3 символа в строках ">" будет отрезаться часть строки до следующего символа.
	gara>18.14.22>helloWorld>alaxCnami;
	return: helloWorld

- `-d <разделитель>`: Указывает символ-разделитель для полей.
- `-f <поля>`: Определяет номера полей для извлечения. Можно указать одно поле (`-f 1`), диапазон (`-f 1-3`) или несколько полей через запятую (`-f 1,5`). 
- `-b <байты>`: Извлекает данные по диапазону байтов. 
- `-c <символы>`: Извлекает данные по диапазону символов.

1. Извлечение первого поля из файла `/etc/passwd`:
    
    - Синтаксис: `cut -d : -f 1 /etc/passwd`
    - Описание: Команда выведет список всех пользователей системы, поскольку первое поле в `/etc/passwd` — это имя пользователя. 
    
2. 2. **Извлечение первых трех символов из каждой строки файла**:
    
    - Синтаксис: `cut -c 1-3 имя_файла.txt`
    - Описание: Будут выведены только первые три символа из каждой строки входного файла.

## grep - найти в строке
```
$ grep <options> findMy <file.txt>
```
Она используется для поиска определённых слов, фраз или шаблонов в текстовых файлах

### Options:

`-r`указывается ****должен быть рекурсивным**** , то есть поиск будет осуществляться также и во всех подпапках.

Опция -i позволяет искать строку без учёта регистра.

Отображение количества совпадений 
```
grep -c "unix" geekfile.txt
```

отобразить файлы, содержащие заданную строку/шаблон. 
```
grep -l "unix" *

grep -l "unix" f1.txt f2.txt f3.xt f4.txt
```

Проверка целых слов 
```
grep -w "unix" geekfile.txt
```

`-q` тихий режим - (не выводит результат, только возвращает код) 0 или 1

`-e` регулярное выражение
```
# Ищем строки содержащие "error" ИЛИ "warning"
grep -e "error" -e "warning" logfile.txt

# Без -e команда сломается, т.к. --version воспримется как флаг grep
grep -e "--version" config.txt

# Поиск "error" или "fail" без учета регистра
grep -i -e "error" -e "fail" application.log
```

`-E` расширенное регулярное выражение (включает метасимволы | + ? () {} )  
```
# Ищем строки с "success" ИЛИ "completed" ИЛИ "done"
grep -E "success|completed|done" status.log

# Ищем "color" или "colour"
grep -E "colou?r" text.txt

# Ищем числа в формате XXX-XX-XX (одна или больше групп цифр-дефис)
grep -E "([0-9]+-)+[0-9]+" data.txt

# Ищем телефон в формате XXX-XXX-XXXX
grep -E "[0-9]{3}-[0-9]{3}-[0-9]{4}" contacts.txt
```

## cp - скопировать файл или каталог

```
cp <путь к файлу> <название файла-копии или путь к каталогу копирования>
```

`-r`— скопировать каталоги рекурсивно (включая вложенные каталоги)

`-i` — предупредить, если в результате копирования будет перезаписан существующий файл

## mv - переместить или переименовать файл или каталог


```
mv <путь к файлу> <новое название файла или путь к каталогу перемещения>
```

`-i` — предупредить о перезаписи существующего файла

`-n` — не заменять существующий файл
## rm - удалить файл или каталог

```
rm <путь к файлу или каталогу>
```

`-r`— удалить объект рекурсивно (включая содержимое вложенных каталогов)

`-i` — запрашивать согласие на удаление каждого объекта

`-f` — принудительно удалить указанный объект
