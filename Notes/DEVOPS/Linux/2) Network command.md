## id addr

```
$ ip addr
```

Отобразить все сетевые интерфейсы и их адреса.

- Отображение информации для определенного интерфейса (например, `eth0`):

```
$ ip addr show eth0
```

- **Отображать только адреса IPv4 для определенного интерфейса:**

```
$ ip -4 addr show eth0
```

## ping

```
$ ping <параметр> 8.8.8.8
```

Параметры:
- **-4** - использовать только ipv4 (по умолчанию);
- **-6** - использовать только ipv6;
- **-A** - адаптивный режим, время между отправками пакета адаптируется к времени передачи и приема пакета, но не меньше чем 200мс;
- **-b** - разрешить ping широковещательного адреса;
- **-с** - количество пакетов, которые нужно отправить;
- **-D** - выводить время в виде UNIX timestamp;
- **-f** - режим флуда, в этом режиме пакеты передаются без задержек, может использоваться для совершения DoS атак на отдельные узлы. Количество точек, которые выводит утилита обозначает количество потерянных пакетов;
- **-i** - интервал в секундах между отправкой пакетов;
- **-I** - использовать этот сетевой интерфейс для отправки пакетов;
- **-l** - режим перегрузки, отправляется очень много пакетов и система не следит за ответными пакетами;
- **-n** - не получать домены для ip адресов;
- **-r** - игнорировать таблицы маршрутизации и отправить пакет на указанный интерфейс;
- **-s** - размер одного пакета;
- **-t** - установить TTL вручную;
- **-v** - более подробный вывод.
## ?! whois, dig, nslookup - домены и IP

```
$ whois google.com
```
==Команда whois== — способная предоставить подробную информацию о домене или IP-адресе.

| Флаг | Описание                                         |
| ---- | ------------------------------------------------ |
| `-H` | Скрыть правовые оговорки.                        |
| `-i` | Включить поиск без учета регистра.               |
| `-l` | Вывод в одну строку. Полезно для скриптов.       |
| `-p` | Укажите номер порта.                             |
| `-r` | Отключить рекурсивный поиск.                     |
| `-R` | Покажите сырой, необработанный ответ.            |
| `-T` | Укажите типы запросов.                           |
| `-v` | Подробный вывод.                                 |
| `-x` | Отображать дополнительную отладочную информацию. |
| `-B` | Не удаляйте непечатаемые символы.                |
| `-G` | Отключите хаки, специфичные для Google.          |
| `-K` | Отключить проверку известных серверов.           |
| `-P` | Отключить преобразование punycode.               |
| `-d` | Верните подробную информацию.                    |

==Команда dig ==— это гибкий инструмент для опроса DNS-серверов. Она выполняет DNS-поиск и отображает ответы, полученные от запрошенных DNS-серверов.
```
$ dig google.com
```

==Команда nslookup== — это инструмент, позволяющий запрашивать информацию в системе доменных имён (DNS) для сопоставления доменного имени или IP-адреса.
```
$ nslookup google.com
``` 
## traceroute - маршрут пакета
==Command Traceroute== — это утилита командной строки Linux, используемая для отслеживания пути сетевых пакетов от источника до конечного хоста или IP-адреса.
```
$ traceroute <параметр> <domain/IP-addr>
```
- **-n —** отключить поиск имени хоста и отображать только IP-адреса
- **-w** для изменения времени ожидания ответа
- **-m** для указания максимального TTL (максимального количества переходов)
### Описание

Команда traceroute выполняет свою диагностическую функцию, отправляя эхо-пакеты ICMP, также известные как ping-пакеты, с увеличивающимся значением TTL (времени жизни). Изначально значение TTL равно 1, поэтому первый маршрутизатор получает пакет, уменьшает его до 0 и отбрасывает его, отправляя в ответ сообщение «Время ICMP превышено».

В дальнейшем traceroute увеличивает значение TTL для каждого отправляемого пакета. Это означает, что следующий маршрутизатор, получивший эти пакеты, уменьшает значение TTL до 0 и также возвращает сообщение об истечении времени. Это продолжается до тех пор, пока пакет не достигнет тестируемого узла назначения. По пути traceroute регистрирует каждый переход и предоставляет имя хоста, IP-адрес и время прохождения туда и обратно для каждого из них.

Это позволяет составить карту пути, по которому пакеты достигают пункта назначения, давая нам более точное представление о маршруте. Программа определяет все маршрутизаторы на пути, что помогает выявить задержки в сети и узкие места, а иногда даже неисправные устройства, на которых пакеты просто останавливаются.
