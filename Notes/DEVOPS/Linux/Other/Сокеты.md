# Command
`$ sudo ss -ntlp или -ntulp` - **list** *tcp* or *tcp and udp* sockets
# Теория
Сокет — это труба, буфер памяти в файловой системе, через который один процесс может передавать данные другому. Только вместо соединения двух процессов на твоём компьютере, сокет соединяет процесс на твоём компьютере с процессом на компьютере _кого-то другого_ через сеть.

Как и в трубе, есть система, с помощью которой пишущий процесс может записывать данные в трубу и сигнализировать о наличии данных; читающий процесс получает сигнал о готовности данных к чтению и может их прочитать. Трубы однонаправленные, но процесс может подключаться к нескольким трубам, так что двунаправленный поток данных может осуществляться просто с помощью двух труб в противоположных направлениях.

Если объединить двунаправленную трубу со средством, с помощью которого можно _адресовать_ другой компьютер и получить доступ к "общедоступной трубе", размещённой на нём, то по сути вы получите сокет. С "общедоступной" стороны сокет имеет вид _порта_, нумерованного субадреса, который однозначно идентифицирует сокет; с внутренней, ориентированной на процесс стороны, сокет имеет вид дескриптора файла, как и трубы.

```bash
127.0.0.1:9090  # подключиться без махинаций не выйдет

0.0.0.0:80      # слушается на всех интерфейсах, которые есть в системе, такие неосходимо закрывать
```
