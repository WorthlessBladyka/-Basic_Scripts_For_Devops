## module "command" and "copy"
![[Снимок экрана 2025-11-14 в 6.11.48 PM.png]]
1) module "command"
	- Параметр ==chdir== гарантирует, что Ansible изменит каталог на /etc перед выполнением команды
	- Параметр ==creates== используется для проверки перед запуском команды. Например, эта команда для создания папки будет запущена только в том случае, если папки не существует.
	- ==free_form== указывает, что этот модуль принимает для запуска команду произвольной формы
		- Ввод команды, такой как `cat /etc/resolv.conf` или `mkdir /folder`, является вводом free_form. Не все модули поддерживают такой ввод.
2) Module "copy", который используется для копирования файлов из источника (src) в место назначения (dest).
	- Он принимает только параметризованный ввод, а не ввод произвольной формы(free-form)

## Module service
![[Снимок экрана 2025-11-14 в 6.23.36 PM.png]]
- Модуль `service` используется для менеджмента служб в управляемой системе, таких как запуск, остановка или перезапуск сервиса.
- Не имеет ввода free_form
- Есть два способа написать подобные утверждения: мы можем написать это в виде параметр = значение (слева) или записать в формате dictionary или map (справа).
