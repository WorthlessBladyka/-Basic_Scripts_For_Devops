# Сети
![[Снимок экрана 2025-10-29 в 5.52.54 PM.png]]
Базовые(устанавливаются по умолчанию):
- Bridge - (по умол.) мостовая сеть
- None - не имеет доступа к внешней сети
- Host - отключает преимущества NET и располагает контейнер на host-овой сети
Также:
- Overlay - этот тип сети адаптирован для работы на одной сети для нескольких узлах
- Macvlan - интересна тем, кто будет docker-изировать нагрузки связанные с виртуальными машинами, эмулирует полностью физический хост части сети
- 3rdPARTY_PLUGINS - кастомные сети, узко-специфичные

# DNS
- Docker имеет встроенный DNS-server
- При связывании разных микро-сервисов рекомендовано использовать имя сервиса, а не его ip
```
$ mysql.connect(127.12.0.5)      #connect по ip
$ mysql.connect(mysql-db)        #connect по name
```
- Встроенный DNS всегда работает по адресе 127.0.0.11
![[Снимок экрана 2025-10-29 в 6.00.48 PM 1.png]]
- ТО есть, каждый docker-counter используется свое пространство имен(namespace), которое позволяет разному трафику обрабатываться и передаваться по разным правилам, делать это рядом друг с другом и не заметно друг для друга.
- Так же используется виртуальное ethernet соединение и их интерфейсы для связывание контейнеров вместе

# Command
``` check_list_network
$ docker network ls 
			# просмотреть доступные соединения
```

``` create_new_network
$ docker network create --driver bridge --subnet 192.0.0.0/24 --gateway 192.0.0.1 name_network
```

``` run_--network
$ docker run --network=my_custom_net webserver/image
			# создать контейнер и определить сеть
```

``` check_port_image
$ docker image inspect name_image
```
