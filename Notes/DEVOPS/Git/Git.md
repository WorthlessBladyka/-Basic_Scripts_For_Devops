# Настройки

Перед началом работы нужно выполнить некоторые настройки:
```
git config --global user.name "Your Name" # указать имя, которым будут подписаны коммиты
git config --global user.email "e@w.com"  # указать электропочту, которая будет в описании коммитера
```

Если вы в Windows:
```
git config --global core.autocrlf true # включить преобразование окончаний строк из CRLF в LF
```

SSH
```bash
		# login git ssh
~/.ssh/config

HostName github.com

IdentityFile ~/.ssh/new_git_ssh_1.txt
IdentitiesOnly yes
```

# clone and pull
`git clone` создает _полную копию_ существующего удаленного репозитория на вашем компьютере с его историей и ветками, а `git pull` _скачивает новые изменения_ (коммиты) из уже существующего локального репозитория, который связан с удаленным, и _автоматически сливает_ их с вашей текущей веткой, по сути, это связка `fetch` и `merge`. `clone` используется для начала работы с проектом, а `pull` — для синхронизации уже локального репозитория с сервером.
```shell
git pull origin            # влить изменения с удалённого репозитория 
							(все ветки)
git pull origin master     # влить изменения с удалённого репозитория 
							(только указанная ветка)
```
```shell
		# клонировать репозиторий в текущую директорию
git clone https://github.com:nicothin/web-design.git .
```

# Индекс
```shell
		# Добавление в индекс
git add .        # добавить в индекс все новые, изменённые, удалённые файлы из текущей директории и её поддиректорий
git add text.txt # добавить в индекс указанный файл (был изменён, был удалён или это новый файл)
git add -i       # запустить интерактивную оболочку для добавления в индекс только выбранных файлов
git add -p       # показать новые/изменённые файлы по очереди с указанием их изменений и вопросом об отслеживании/индексировании
```

```shell
		# Удаление изменений из индекса
git reset
```
