## Как работает ROS. Что такое package, node, message, издатель, подписчик.

ROS (Robot operation system) - узлы в ROS - это вычислительные процессы. Каждый узел выполняет обработку определенных данных. Узлы в ROS создаются с помощью клиентской библиотеки (например: rospy or roscpp).

### Master_node
 
 Все описанное выше контролируется master node. Он выполняет роль сервера мастера имен для возможности подключения между собой различных node. 
 Команда rescore запускает сервер мастера. При запуске мастер будет запущен по URL-адресу, установленной в переменной окружения ROS_MASTER_URL.

![[Снимок экрана 2025-09-29 в 11.53.41 PM.png]]

### Message

Message - node отправляют и принимают данные между собой, согласно заданого формата. Эти данные называются "message", а описание "типом сообщений".
Сообщения могут быть как простых типов (int, bool, float), так и могут состоять из сложных структур

![[Снимок экрана 2025-09-29 в 11.59.03 PM.png]]

### Package (пакет)

Пакеты являются основной единицей ROS. Любое приложение в ROS оформляется в пакет, в котором определяется: config package, ноды необходимые для работы пакета, зависимости от других пакетов.
> Грубо говоря, node - это исполняемый файл (программы python и cpp и тд). Несколько нод для одной задачи составляют пакет. Помимо нод в пакет входят и другие составляющие
